## ADDED Requirements

### Requirement: Referências sobre Spec-Driven Development e Test Harness
O research SHALL produzir referências que cubram o conceito de Spec-Driven Development (escrever spec antes do código), Test Harness como ambiente para IA gerar código mais seguro, e geração de suites de teste com IA.

#### Scenario: Referências sobre SDD e TDD com IA
- **WHEN** o agente pesquisa Spec-Driven Development e Test-Driven Development com IA
- **THEN** o output MUST incluir pelo menos 3 referências sobre a prática de escrever especificações/testes antes da implementação, com exemplos práticos

#### Scenario: Referências cobrem geração de testes com IA
- **WHEN** o agente busca por geração automática de testes com IA
- **THEN** o output MUST incluir pelo menos 2 referências sobre como usar IA para gerar unitários, testes de integração e edge cases de forma confiável

### Requirement: Referências sobre code review eficiente com IA
O research SHALL identificar materiais sobre como conduzir code reviews mais eficientes com IA — tanto para revisar código gerado por IA quanto para usar IA como assistente no processo de review.

#### Scenario: Referências sobre checklist de PR e code review com IA
- **WHEN** o agente pesquisa code review com IA
- **THEN** o output MUST incluir pelo menos 2 referências com checklists ou frameworks de PR focados em segurança, performance e regressões

#### Scenario: Referências sobre como revisar criticamente código gerado por IA
- **WHEN** o agente busca por boas práticas de revisão de código de IA
- **THEN** o output MUST incluir pelo menos 1 referência com abordagem estruturada para revisão crítica de outputs de IA em código
